Linear Search

Linear search scans an array from the beginning, comparing each element to the search key in turn. It works on unsorted and sorted arrays alike. If the key is found, its index is returned; otherwise −1 is returned.

LinearSearchDemo.java
import java.util.Scanner;

public class LinearSearchDemo {
    private static int linearSearch(int[] numbers, int key) {
        for (int i = 0; i < numbers.length; i++) {
            if (numbers[i] == key) {
                return i;
            }
        }
        return -1; // not found
    }

    public static void main(String[] args) {
        int[] numbers = {2, 4, 7, 10, 11, 32, 45, 87};
        Scanner scnr = new Scanner(System.in);

        System.out.print("Enter an integer: ");
        int key = scnr.nextInt();
        int idx = linearSearch(numbers, key);

        if (idx == -1) {
            System.out.println(key + " was not found.");
        } else {
            System.out.println("Found " + key + " at index " + idx + ".");
        }
    }
}

Time complexity: O(n) — in the worst case, every element is checked.

Binary Search

Binary search works only on sorted arrays but is dramatically faster. It checks the middle element and eliminates half the remaining elements with each comparison.

BinarySearchDemo.java
import java.util.Scanner;

public class BinarySearchDemo {
    private static int binarySearch(int[] numbers, int key) {
        int low  = 0;
        int high = numbers.length - 1;

        while (high >= low) {
            int mid = (high + low) / 2;

            if (numbers[mid] < key) {
                low = mid + 1;       // key is in upper half
            } else if (numbers[mid] > key) {
                high = mid - 1;      // key is in lower half
            } else {
                return mid;          // found
            }
        }
        return -1; // not found
    }

    public static void main(String[] args) {
        int[] numbers = {2, 4, 7, 10, 11, 32, 45, 87};
        Scanner scnr = new Scanner(System.in);

        System.out.print("Enter an integer: ");
        int key = scnr.nextInt();
        int idx = binarySearch(numbers, key);

        if (idx == -1) {
            System.out.println(key + " was not found.");
        } else {
            System.out.println("Found " + key + " at index " + idx + ".");
        }
    }
}

Time complexity: O(log n) — each step halves the search space. For 1,000 elements, at most 10 comparisons are needed.

Linear vs Binary Search Comparison

  • Linear search: works on any array (sorted or not), O(n). Use for small arrays or when the data is unsorted.
  • Binary search: requires a sorted array, O(log n). Dramatically faster for large datasets.

Java's Arrays.binarySearch() in java.util.Arrays provides a built-in binary search you can use in practice:

Using the built-in Arrays.binarySearch()
import java.util.Arrays;

int[] sorted = {2, 4, 7, 10, 11, 32, 45, 87};
int idx = Arrays.binarySearch(sorted, 10);
System.out.println("Index: " + idx);  // 3
