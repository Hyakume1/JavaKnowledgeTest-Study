Linear Search — Checking Every Item

Linear search is the simplest way to find something in a list: start at the beginning and check each item one by one until you find what you're looking for (or reach the end). It works on any list — sorted or not — but it can be slow for large lists because you might have to check every single item.

LinearSearchDemo.java
import java.util.Scanner;

public class LinearSearchDemo {
    private static int linearSearch(int[] numbers, int key) {
        for (int i = 0; i < numbers.length; i++) {
            if (numbers[i] == key) {
                return i;   // found it! return its position
            }
        }
        return -1; // checked everything — not found
    }

    public static void main(String[] args) {
        int[] numbers = {2, 4, 7, 10, 11, 32, 45, 87};
        Scanner scnr = new Scanner(System.in);

        System.out.print("Enter an integer: ");
        int key = scnr.nextInt();
        int idx = linearSearch(numbers, key);

        if (idx == -1) {
            System.out.println(key + " was not found.");
        } else {
            System.out.println("Found " + key + " at index " + idx + ".");
        }
    }
}

In the worst case, linear search has to check every item — so for a list of 1,000 items, it could take up to 1,000 checks.

Binary Search — Using a Sorted List to Go Much Faster

Binary search only works on a sorted list, but it's dramatically faster. The idea: check the middle item. If it's too high, search the lower half. If it's too low, search the upper half. Each step cuts the remaining items in half.

Think of it like opening a phone book to find a name — you open to the middle, see you're too far, flip to the middle of the first half, etc. You never read every page.

BinarySearchDemo.java
import java.util.Scanner;

public class BinarySearchDemo {
    private static int binarySearch(int[] numbers, int key) {
        int low  = 0;
        int high = numbers.length - 1;

        while (high >= low) {
            int mid = (high + low) / 2;

            if (numbers[mid] < key) {
                low = mid + 1;       // key must be in the upper half
            } else if (numbers[mid] > key) {
                high = mid - 1;      // key must be in the lower half
            } else {
                return mid;          // found it!
            }
        }
        return -1; // not found
    }

    public static void main(String[] args) {
        int[] numbers = {2, 4, 7, 10, 11, 32, 45, 87};
        Scanner scnr = new Scanner(System.in);

        System.out.print("Enter an integer: ");
        int key = scnr.nextInt();
        int idx = binarySearch(numbers, key);

        if (idx == -1) {
            System.out.println(key + " was not found.");
        } else {
            System.out.println("Found " + key + " at index " + idx + ".");
        }
    }
}

For a list of 1,000 items, binary search needs at most 10 checks. For 1,000,000 items, it needs at most 20 checks. That's the power of cutting the problem in half each time.

When to Use Each

  • Linear search: works on any list (sorted or unsorted). Use for small lists or when the data isn't sorted.
  • Binary search: requires a sorted list, but much faster for large data. Always sort first if you need to search many times.

Java's Built-In Binary Search

Java gives you a built-in binary search so you don't have to write one yourself:

Using the built-in Arrays.binarySearch()
import java.util.Arrays;

int[] sorted = {2, 4, 7, 10, 11, 32, 45, 87};
int idx = Arrays.binarySearch(sorted, 10);
System.out.println("Index: " + idx);  // 3
