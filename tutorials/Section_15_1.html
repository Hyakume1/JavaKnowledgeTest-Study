PrintStream and System.out

A PrintStream provides methods for writing text to an output destination such as the screen or a file. System.out is a pre-created PrintStream that writes to the console. You never need to import or instantiate it — it is always available.

The two main output methods are print() (no newline) and println() (adds a newline). printf() provides formatted output and is covered in Section 15.2.

Basic output
System.out.println("Hello, World!");   // prints and moves to next line
System.out.print("Enter a number: ");  // prints without newline
System.out.println(42);                // works with int, double, boolean, etc.
System.out.println("Sum: " + (3 + 4)); // string concatenation: "Sum: 7"
Output 
Hello, World!
Enter a number: Sum: 7

Scanner and System.in

System.in is a pre-created InputStream connected to the keyboard. Reading from it directly is cumbersome, so Java provides the Scanner class to parse typed input into useful types.

Import java.util.Scanner and create a Scanner wrapping System.in:

Reading user input with Scanner
import java.util.Scanner;

public class InputDemo {
    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);

        System.out.print("Enter your name: ");
        String name = scnr.next();          // reads one word

        System.out.print("Enter your age: ");
        int age = scnr.nextInt();           // reads an integer

        System.out.print("Enter a GPA: ");
        double gpa = scnr.nextDouble();     // reads a double

        System.out.println("Hello " + name + ", age " + age + ", GPA " + gpa);
    }
}
Sample run 
Enter your name: Alex
Enter your age: 20
Enter a GPA: 3.8
Hello Alex, age 20, GPA 3.8

Common Scanner Methods

  • next() — reads the next whitespace-delimited token as a String
  • nextLine() — reads the rest of the current line (including spaces)
  • nextInt() — reads the next token as an int
  • nextDouble() — reads the next token as a double
  • hasNextInt() — returns true if the next token can be parsed as an int

Mixing nextInt() and nextLine()

A common pitfall: nextInt() does not consume the newline character. Calling nextLine() immediately after reads an empty string. Fix by calling scnr.nextLine() once as a "consume the newline" step.

Correct way to mix nextInt and nextLine
int age = scnr.nextInt();
scnr.nextLine();                    // consume the leftover newline
String fullName = scnr.nextLine();  // now reads correctly

IOException and the throws Clause

Operations that involve external I/O (files, network) can fail unpredictably. Java requires you to acknowledge this by adding throws IOException to main() when using raw streams like FileInputStream. The Scanner class handles most of this for keyboard input, so throws IOException is not needed when only reading from System.in.
