Printing to the Screen

You already know System.out.println() — it prints something and then moves to the next line. Its sibling, System.out.print(), prints without moving to a new line. Think of println as pressing Enter after you type, and print as staying on the same line.

System.out is a special object Java gives you automatically — you never need to import or create it. It's always ready to use.

Basic output
System.out.println("Hello, World!");   // prints and moves to next line
System.out.print("Enter a number: ");  // prints WITHOUT moving to next line
System.out.println(42);                // works with numbers too
System.out.println("Sum: " + (3 + 4)); // "Sum: 7"
Output 
Hello, World!
Enter a number: Sum: 7

Reading Input from the User

To read what the user types, Java uses something called a Scanner. Think of Scanner like a helper that sits and waits for the user to type something, then hands the typed value back to your program.

You need to import it at the top of your file and create one Scanner that reads from the keyboard (System.in):

Reading user input with Scanner
import java.util.Scanner;

public class InputDemo {
    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);

        System.out.print("Enter your name: ");
        String name = scnr.next();          // reads one word

        System.out.print("Enter your age: ");
        int age = scnr.nextInt();           // reads a whole number

        System.out.print("Enter a GPA: ");
        double gpa = scnr.nextDouble();     // reads a decimal number

        System.out.println("Hello " + name + ", age " + age + ", GPA " + gpa);
    }
}
Sample run 
Enter your name: Alex
Enter your age: 20
Enter a GPA: 3.8
Hello Alex, age 20, GPA 3.8

Scanner Method Cheat Sheet

  • next() — reads the next single word (stops at a space)
  • nextLine() — reads the entire line, including spaces
  • nextInt() — reads a whole number (like 42)
  • nextDouble() — reads a decimal number (like 3.14)
  • hasNextInt() — checks if the next thing typed is a whole number (returns true or false)

Watch Out: Mixing nextInt() and nextLine()

Here is a classic trap beginners fall into. When you call nextInt(), it reads the number but leaves an invisible "Enter" key sitting in the buffer. If you then call nextLine(), it reads that leftover "Enter" and gives you an empty string instead of the user's input.

The fix: call scnr.nextLine() once right after nextInt() just to throw away that leftover Enter, before you call nextLine() for real.

How to fix the nextInt + nextLine problem
int age = scnr.nextInt();
scnr.nextLine();                    // throw away the leftover Enter
String fullName = scnr.nextLine();  // now this works correctly

Why Does File Input Need throws IOException?

When your program reads from a file (instead of the keyboard), things can go wrong that are out of your control — the file might not exist, or the disk might fail. Java requires you to acknowledge this by adding throws IOException to your main() method when you use file streams. This is Java's way of saying "I know something might go wrong here."

When reading from the keyboard with Scanner and System.in, you do NOT need throws IOException — Scanner handles that for you.
