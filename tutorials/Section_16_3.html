When a File Doesn't Exist

When you try to open a file for reading, that file might not actually be there. If that happens, Java throws a FileNotFoundException. Instead of letting your program crash, you can catch it and print a helpful message to the user.

Handling a missing input file
import java.io.FileInputStream;
import java.io.FileNotFoundException;
import java.util.Scanner;

public class SafeFileOpen {
    public static void main(String[] args) {
        try {
            FileInputStream fis = new FileInputStream("data.txt");
            Scanner inFS = new Scanner(fis);
            System.out.println("Opened successfully.");
            // ... read data ...
            inFS.close();
        } catch (FileNotFoundException e) {
            System.out.println("Error: file not found — " + e.getMessage());
        }
    }
}

Always Close Your Files

An open file takes up resources from your operating system. Just like you should close apps on your phone to save battery, you should always close files in Java when you're done with them. The cleanest way to guarantee this is shown below — declare the file variables before the try block so they can be closed in a finally block no matter what happens.

FileDataAverage.java — read input, write average, close both files
import java.util.Scanner;
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.PrintWriter;
import java.io.FileNotFoundException;

public class FileDataAverage {
    public static void main(String[] args) {
        Scanner   fileScanner = null;
        PrintWriter fileWriter  = null;

        try {
            fileScanner = new Scanner(new FileInputStream("input.txt"));
            fileWriter  = new PrintWriter(new FileOutputStream("output.txt"));

            int count = fileScanner.nextInt();
            int sum   = 0;
            for (int i = 0; i < count; i++) {
                sum += fileScanner.nextInt();
            }
            fileWriter.println("Average: " + (double) sum / count);

        } catch (FileNotFoundException e) {
            System.out.println("Could not open a file: " + e.getMessage());
        } finally {
            if (fileScanner != null) fileScanner.close();
            if (fileWriter  != null) fileWriter.close();
        }
    }
}
input.txt → output.txt 
input.txt: 3  3  2  1
output.txt: Average: 2.0

The finally Block

A finally block runs no matter what — even if an exception was thrown, even if the try block ran perfectly. This makes it the perfect place to put cleanup code like closing files. It's like saying "no matter what happens, always do this at the end."

finally always runs
try {
    // risky code
    int x = Integer.parseInt("abc"); // this will throw NumberFormatException
} catch (NumberFormatException e) {
    System.out.println("Caught: " + e.getMessage());
} finally {
    System.out.println("This always runs.");
}
Output 
Caught: For input string: "abc"
This always runs.

try-with-resources (The Easy Way)

Java has a cleaner way to handle file closing automatically: try-with-resources. If you declare the file inside the parentheses of try(...), Java will close it automatically when the block ends — no finally needed. This is the recommended modern approach.

try-with-resources — file closes automatically
import java.io.FileInputStream;
import java.io.FileNotFoundException;
import java.util.Scanner;

public class AutoCloseDemo {
    public static void main(String[] args) {
        try (Scanner inFS = new Scanner(new FileInputStream("data.txt"))) {
            while (inFS.hasNextInt()) {
                System.out.println(inFS.nextInt());
            }
        } catch (FileNotFoundException e) {
            System.out.println("File not found.");
        }
        // inFS is automatically closed here — no need to call close()
    }
}
