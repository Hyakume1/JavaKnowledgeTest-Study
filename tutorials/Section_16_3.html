FileNotFoundException

When opening a file for reading, the file may not exist. FileInputStream's constructor throws FileNotFoundException in that case. Catching it lets your program handle the missing file gracefully instead of crashing.

Handling a missing input file
import java.io.FileInputStream;
import java.io.FileNotFoundException;
import java.util.Scanner;

public class SafeFileOpen {
    public static void main(String[] args) {
        try {
            FileInputStream fis = new FileInputStream("data.txt");
            Scanner inFS = new Scanner(fis);
            System.out.println("Opened successfully.");
            // ... read data ...
            inFS.close();
        } catch (FileNotFoundException e) {
            System.out.println("Error: file not found — " + e.getMessage());
        }
    }
}

Always Close Your Files

An open file holds operating-system resources. A good practice is to close every file after use. The safest pattern is to declare file objects before the try block (so they're visible in the finally block) and close them after.

FileDataAverage.java — read input, write average, close both files
import java.util.Scanner;
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.PrintWriter;
import java.io.FileNotFoundException;

public class FileDataAverage {
    public static void main(String[] args) {
        Scanner   fileScanner = null;
        PrintWriter fileWriter  = null;

        try {
            fileScanner = new Scanner(new FileInputStream("input.txt"));
            fileWriter  = new PrintWriter(new FileOutputStream("output.txt"));

            int count = fileScanner.nextInt();
            int sum   = 0;
            for (int i = 0; i < count; i++) {
                sum += fileScanner.nextInt();
            }
            fileWriter.println("Average: " + (double) sum / count);

        } catch (FileNotFoundException e) {
            System.out.println("Could not open a file: " + e.getMessage());
        } finally {
            if (fileScanner != null) fileScanner.close();
            if (fileWriter  != null) fileWriter.close();
        }
    }
}
input.txt → output.txt 
input.txt: 3  3  2  1
output.txt: Average: 2.0

try-with-resources (Recommended)

Java's try-with-resources statement automatically closes any resource declared in the try(...) parentheses when the block exits — whether normally or due to an exception. No finally block needed.

try-with-resources — files close automatically
import java.io.FileInputStream;
import java.io.FileNotFoundException;
import java.util.Scanner;

public class AutoCloseDemo {
    public static void main(String[] args) {
        try (Scanner inFS = new Scanner(new FileInputStream("data.txt"))) {
            while (inFS.hasNextInt()) {
                System.out.println(inFS.nextInt());
            }
        } catch (FileNotFoundException e) {
            System.out.println("File not found.");
        }
        // inFS is automatically closed here
    }
}

The finally Block

A finally block always runs when the try block exits — regardless of whether an exception was thrown or caught. Use it to guarantee cleanup code (like closing files or database connections) always executes.

finally always runs
try {
    // risky code
    int x = Integer.parseInt("abc"); // throws NumberFormatException
} catch (NumberFormatException e) {
    System.out.println("Caught: " + e.getMessage());
} finally {
    System.out.println("This always runs.");
}

Caught: For input string: "abc"
This always runs.
