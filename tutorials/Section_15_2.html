What Is printf()?

Sometimes println() isn't enough. Imagine you want to print a table where all the numbers line up in neat columns, or you want to show a price as exactly $3.50 instead of $3.5. That's what printf() is for — it lets you control exactly how your output looks.

System.out.printf() works by giving it a format string. The format string is like a template with blank spots (called format specifiers) that start with %. Java fills in each blank with the values you provide after the format string, in order.

Format specifiers fill in the blanks in order
String name = "Alice";
double gpa  = 3.5;
int age     = 20;

System.out.printf("%d - %s", age, name);
Output 
20 - Alice

The first %d grabs age (an int → 20), and %s grabs name (a String → "Alice"). They fill in left to right.

Format Specifier Cheat Sheet

  • %d — a whole number (int)
  • %f — a decimal number (double) — by default shows 6 decimal places
  • %s — a String (any text)
  • %% — a literal percent sign % — because a single % always starts a format specifier!
  • %n — a newline (like pressing Enter)

The %% trap: If you want to print an actual % sign, you must write %%. A single % tells Java "a format specifier is coming" — so % alone is never a literal percent.

Printing a literal percent sign
String name = "Alice";
double gpa  = 3.5;
int age     = 20;

System.out.printf("%f%% %d\n", gpa, age);
Output 
3.500000% 20

%f prints gpa (3.5 → 3.500000), %% prints a literal %, then a space, then %d prints age (20).

Controlling Decimal Places

Put a dot and a number before f to set how many decimal places to show. For example, %.2f means "show exactly 2 decimal places." You can also add a width before the dot: %8.2f means "8 total characters, 2 decimal places."

Decimal number formatting
System.out.printf("%.2f miles\n", 10.3);  // → 10.30 miles

Aligning Numbers in Columns

Put a width number before d to reserve that many characters. Numbers right-align by default. A minus sign (%-10s) left-aligns text.

Right-aligning integers in a column
System.out.printf("%d, %d\n", age, age + 1);
Output (age = 20) 
20, 21

Buffering — When Does Output Actually Appear on Screen?

This is a tricky but important concept. When your code calls print() or printf(), the characters do NOT necessarily appear on screen right away. They go into a buffer — like a waiting room — and sit there until something triggers a flush.

Here are the rules for System.out (which has autoFlush = true):

  • println() → always flushes immediately ✓
  • print() with \n in the string → also flushes (autoFlush detects the newline) ✓
  • flush() → forces everything buffered to appear right now ✓
  • print() with NO newline → buffered, does NOT appear yet ✗
Tracing exactly which lines appear immediately
// Poem by Henry David Thoreau
System.out.print("My life has been the poem ");  // A — buffered (no newline)
System.out.println("I would have writ");          // B — flushed immediately ← println
System.out.print("But I could not ");             // C — buffered again
System.out.flush();                               // explicit flush — C appears now
System.out.print("both live and utter it.\n");    // D — flushed ← \n triggers autoFlush

Summary: Line A waits. Line B flushes (and also sends A with it since it was buffered). Line C waits. flush() sends C. Line D flushes itself because the string contains \n.
