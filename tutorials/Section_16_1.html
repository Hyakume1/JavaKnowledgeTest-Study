What Is an Exception?

Imagine you're following a recipe, and it says "add 2 eggs." But you open the fridge and there are no eggs. The recipe can't continue normally — something unexpected happened. In Java, that unexpected problem is called an exception.

An exception is an error that happens while your program is running. Common examples:

  • The user types "hello" when your program expects a number
  • Your code tries to divide a number by zero
  • Your code tries to use a variable that has no value (null)

If an exception is not handled, the program stops immediately and prints an ugly error message.

Unhandled exception — program crashes
Scanner scnr = new Scanner(System.in);
int x = scnr.nextInt();  // user types "hello" instead of a number
// → throws InputMismatchException; program ends immediately

Catching Exceptions with try / catch

Instead of letting your program crash, you can catch the exception and handle it gracefully. The idea is simple:

  • Put the risky code inside a try block — "try to do this..."
  • Write what should happen if something goes wrong inside a catch block — "...but if it fails, do this instead."
Basic try-catch structure
try {
    // code that might cause an error
} catch (ExceptionType varName) {
    // code that runs if the error happens
}

Retrying After Bad Input

A really useful pattern: catch the bad input and ask the user to try again using a loop.

LightTravelTime.java — keeps asking until valid input is given
import java.util.Scanner;
import java.util.InputMismatchException;

public class LightTravelTime {
    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);
        double distMiles = 0.0;
        boolean needInput = true;

        while (needInput) {
            System.out.print("Enter a distance in miles: ");
            try {
                distMiles = scnr.nextDouble();
                needInput = false;           // it worked! stop the loop
            } catch (InputMismatchException e) {
                scnr.nextLine();             // clear out the bad input
                System.out.println("Please enter a number.");
            }
        }

        double travelTime = distMiles / 186282.0;
        System.out.printf("Light travels %.1f miles in %.6f seconds.%n",
                          distMiles, travelTime);
    }
}
Sample run 
Enter a distance in miles: hello
Please enter a number.
Enter a distance in miles: 1000000
Light travels 1000000.0 miles in 5.368205 seconds.

Getting the Error Message

Every exception comes with a message describing what went wrong. You can access it by calling .getMessage() on the exception object inside your catch block.

Printing exception details
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Caught: " + e.getMessage()); // "/ by zero"
}
Output 
Caught: / by zero

Common Exception Types You'll See

  • InputMismatchException — Scanner got the wrong type (e.g., text instead of a number)
  • ArithmeticException — illegal math (e.g., dividing by zero)
  • NullPointerException — tried to use a variable that is null (has no value)
  • ArrayIndexOutOfBoundsException — tried to access an array position that doesn't exist
  • NumberFormatException — tried to convert text like "abc" into a number
  • FileNotFoundException — tried to open a file that doesn't exist
