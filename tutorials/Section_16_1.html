What Is an Exception?

An exception is an event that disrupts the normal flow of a program — such as dividing by zero, reading invalid input, or accessing a null reference. If an exception is not handled, the program terminates with an error message.

Unhandled exception — program crashes
Scanner scnr = new Scanner(System.in);
int x = scnr.nextInt();  // user types "hello" instead of a number
// → throws InputMismatchException; program ends immediately

try / catch Blocks

Wrap risky code in a try block. If an exception is thrown, execution jumps to the matching catch block instead of crashing.

Basic try-catch structure
try {
    // code that might throw
} catch (ExceptionType varName) {
    // code to run if that exception occurs
}

Handling an Exception and Retrying

A catch block can also re-ask the user for valid input using a loop.

LightTravelTime.java — retries on bad input
import java.util.Scanner;
import java.util.InputMismatchException;

public class LightTravelTime {
    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);
        double distMiles = 0.0;
        boolean needInput = true;

        while (needInput) {
            System.out.print("Enter a distance in miles: ");
            try {
                distMiles = scnr.nextDouble();
                needInput = false;           // success — exit loop
            } catch (InputMismatchException e) {
                scnr.nextLine();             // discard bad input
                System.out.println("Please enter a number.");
            }
        }

        double travelTime = distMiles / 186282.0;
        System.out.printf("Light travels %.1f miles in %.6f seconds.%n",
                          distMiles, travelTime);
    }
}
Sample run 
Enter a distance in miles: hello
Please enter a number.
Enter a distance in miles: 1000000
Light travels 1000000.0 miles in 5.368205 seconds.

Accessing the Exception Message

Every exception object has a getMessage() method that returns a description of the problem.

Printing exception details
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Caught: " + e.getMessage()); // "/ by zero"
}
Output 
Caught: / by zero

Common Exception Types

  • InputMismatchException — Scanner received the wrong type
  • ArithmeticException — illegal math operation (e.g., divide by zero)
  • NullPointerException — calling a method on a null reference
  • ArrayIndexOutOfBoundsException — array index out of range
  • NumberFormatException — bad string-to-number conversion
  • FileNotFoundException — requested file doesn't exist
