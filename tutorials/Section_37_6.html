Building a Queue With an Array

You can also implement a queue using an array. The tricky part is: when you dequeue, instead of shifting every item to fill the gap, you just move a pointer called frontIndex forward. This leaves a "hole" at the beginning — but instead of wasting it, the queue wraps around and reuses it. This technique is called a circular buffer.

You track two things:

  • frontIndex — where the front of the queue is in the array
  • length — how many items are currently in the queue
Circular buffer concept
// Array size 6. After enqueue(A), enqueue(B), enqueue(C), dequeue():
// frontIndex = 1, length = 2
//  index: [ 0,  1,  2,  3,  4,  5 ]
//  data:  [ A,  B,  C,  -,  -,  - ]
//              ^front

// After enqueue(D), enqueue(E), enqueue(F), enqueue(G) (wraps around):
//  index: [ G,  B,  C,  D,  E,  F ]
//  frontIndex = 1, length = 6

Bounded vs Unbounded

  • A bounded queue has a fixed max size. If it's full and you try to enqueue, you get false back instead of crashing.
  • An unbounded queue doubles its array size when full and copies items over so that frontIndex resets to 0.

Enqueue and Dequeue Logic

The key formula: the next empty slot for a new item is at position (frontIndex + length) % array.length. The % (modulo) operator makes the index wrap around to the beginning when it goes past the end of the array.

Enqueue and dequeue using wrap-around math
// Enqueue: figure out where the new item goes and put it there
int newIndex = (frontIndex + length) % array.length;
array[newIndex] = item;
length++;

// Dequeue: return the front item and move the front pointer forward
T val = array[frontIndex];
frontIndex = (frontIndex + 1) % array.length;
length--;
return val;

Step-by-Step Trace: Bounded Queue with Capacity 4

Watching the queue fill up and wrap around
ArrayQueue q = new ArrayQueue(4);  // max 4 items
q.enqueue("A");   // [A, -, -, -]  front=0 length=1
q.enqueue("B");   // [A, B, -, -]  front=0 length=2
q.enqueue("C");   // [A, B, C, -]  front=0 length=3
q.dequeue();      // returns "A"   [-, B, C, -]  front=1 length=2
q.enqueue("D");   // [-, B, C, D]  front=1 length=3
q.enqueue("E");   // [E, B, C, D]  front=1 length=4  (E wraps to index 0!)
q.enqueue("F");   // returns false — queue is full

Why Wrap Around?

Without wrapping, every dequeue would leave an empty slot at the front that can never be used again — wasting memory. The circular trick lets you reuse those freed-up slots by wrapping the pointer back to the start of the array. The queue never wastes space.
