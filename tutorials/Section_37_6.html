Building a Queue With an Array — Circular Buffer

You can implement a queue using an array. The trick: instead of shifting every item when you dequeue, you just move a pointer called frontIndex forward. When the pointer reaches the end of the array, it wraps back to position 0. This is called a circular buffer.

You track two things:

  • frontIndex — the index of the front (oldest) item in the array
  • length — how many items are currently in the queue

Finding Items in a Circular Queue

Because items can wrap around, calculating their positions requires modulo math:

  • Front item: array[frontIndex]
  • Back item: array[(frontIndex + length - 1) % array.length]
  • Next empty slot (for enqueue): array[(frontIndex + length) % array.length]
Examples: array=[67, 19, 44, 38], size=4
// frontIndex=3, length=2:
//   front = array[3] = 38
//   back  = array[(3+2-1)%4] = array[0] = 67   (wraps around!)

// frontIndex=0, length=3:
//   front = array[0] = 67
//   back  = array[(0+3-1)%4] = array[2] = 44

// frontIndex=1, length=4  (full — all 4 items!):
//   front-to-back order: array[1]=19, array[2]=44, array[3]=38, array[0]=67
//   (wraps: 1 → 2 → 3 → 0)

Enqueue and Dequeue Logic

Enqueue and dequeue using wrap-around math
// Enqueue: insert at (frontIndex + length) % array.length
int newIndex = (frontIndex + length) % array.length;
array[newIndex] = item;
length++;

// Dequeue: return front item and advance frontIndex
T val = array[frontIndex];
frontIndex = (frontIndex + 1) % array.length;
length--;
return val;

Bounded vs Unbounded

  • A bounded queue has a fixed max size. If it's full (length == maxLength) and you try to enqueue, it returns false (or throws an exception). It can genuinely be "full."
  • An unbounded queue resizes when the array is full — so it never truly fills up. maxLength is set to -1 (no limit).

Only a bounded queue can be "full." An unbounded queue always makes room by resizing.

The maxLength variable is only needed for the bounded queue. Unbounded queues don't need a cap.

When enqueue() Returns False

enqueue() returns false when the bounded queue is full: length == maxLength. It cannot add more items because there's no room and it won't resize.

enqueue() with bounded check
boolean enqueue(int item) {
    if (length == maxLength) { return false; }  // bounded and full → reject
    if (length == array.length) { resize(); }   // unbounded and array full → grow
    int newIndex = (frontIndex + length) % array.length;
    array[newIndex] = item;
    length++;
    return true;
}

How resize() Works

resize() doubles the array size and copies all items starting from frontIndex, wrapping around. After copying, frontIndex resets to 0 so the queue is in a clean, non-wrapped state.

resize() traces for array=[73,91,87,62], frontIndex=2, length=4
// newSize = array.length * 2 = 4 * 2 = 8
// Copy starting at frontIndex=2, wrapping:
//   i=0: (2+0)%4 = 2 → array[2] = 87  → newArray[0] = 87
//   i=1: (2+1)%4 = 3 → array[3] = 62  → newArray[1] = 62
//   i=2: (2+2)%4 = 0 → array[0] = 73  → newArray[2] = 73
//   i=3: (2+3)%4 = 1 → array[1] = 91  → newArray[3] = 91
// Result: newArray = [87, 62, 73, 91, -, -, -, -]
// frontIndex resets to 0

If maxLength = 6 (a cap), the new size = min(8, 6) = 6. Otherwise the new size = 8.

// queueA: array=[73,91,87,62], frontIndex=2, length=4, maxLength=-1
// maxLength = -1 → unbounded (no limit)
// length == array.length → array is full, but resize will happen, not rejection
// Therefore: queueA is UNBOUNDED

Complexity of Enqueue and Dequeue

  • Enqueue worst case: O(n) — when the array needs to be resized, it copies all n elements to a new array. That's O(n).
  • Dequeue worst case: O(1) — just adjusts frontIndex and decrements length. Constant work regardless of queue size.

Only enqueue can trigger a resize. Dequeue never modifies the array — it only moves the front pointer.

An alternate implementation could store the back item's index instead of the length — both approaches are valid and equivalent.
