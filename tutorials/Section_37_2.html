Building a Stack With a Linked List

You know what a stack does — now let's see how to build one from scratch using a linked list.

A linked list is made up of nodes, where each node holds a value and a pointer to the next node. For a stack, we keep the "top of stack" at the head (front) of the linked list. Push = add a node to the front. Pop = remove the node from the front. Both operations are instant because you only ever touch the front.

What a Node Looks Like

LinkedListStack node and class outline
public class LinkedListStack<T> {
    private class Node {
        T data;
        Node next;
        Node(T data) { this.data = data; }
    }

    private Node top;    // the front of the linked list = top of the stack
    private int length;

    public LinkedListStack() {
        top    = null;
        length = 0;
    }
    ...
}

Push: Adding to the Top

Create a new node, point it at the current top, then make it the new top. That's it!

push()
public void push(T item) {
    Node newNode = new Node(item);
    newNode.next = top;   // new node points to the old top
    top = newNode;        // new node IS the new top
    length++;
}

Pop: Removing From the Top

Save the top node's data, move the top pointer to the next node, and return the saved data.

pop()
public T pop() {
    if (isEmpty()) throw new RuntimeException("Stack is empty.");
    T data = top.data;   // save the value we're removing
    top = top.next;      // step past the removed node
    length--;
    return data;
}

Full Implementation and Demo

LinkedListStack.java
public class LinkedListStack<T> {
    private class Node {
        T data; Node next;
        Node(T d) { data = d; }
    }

    private Node top;
    private int length;

    public boolean isEmpty()  { return length == 0; }
    public int     getLength(){ return length; }

    public T peek() {
        if (isEmpty()) throw new RuntimeException("Stack is empty.");
        return top.data;
    }

    public void push(T item) {
        Node n = new Node(item);
        n.next = top;
        top = n;
        length++;
    }

    public T pop() {
        if (isEmpty()) throw new RuntimeException("Stack is empty.");
        T val = top.data;
        top   = top.next;
        length--;
        return val;
    }

    public static void main(String[] args) {
        LinkedListStack<Integer> stack = new LinkedListStack<>();
        stack.push(5);
        stack.push(10);
        stack.push(15);
        System.out.println("Peek: " + stack.peek());   // 15
        System.out.println("Pop:  " + stack.pop());    // 15
        System.out.println("Pop:  " + stack.pop());    // 10
        System.out.println("Size: " + stack.getLength()); // 1
    }
}
Output

Size: 1

Linked List vs Array for a Stack

  • A linked list stack can grow as large as you want (as long as your computer has memory).
  • An array-based stack has a fixed max size (unless you resize it), but is a bit faster because of how computers read memory in sequence.
  • Both push and pop are equally fast in either implementation — they both take constant time regardless of stack size.
