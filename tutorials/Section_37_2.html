Implementing a Stack with a Linked List

A stack is naturally implemented using a singly-linked list where the head node always represents the top of the stack. Push prepends a new node; pop removes the head node. Both operations are O(1) — they touch only the head.

Node Structure

LinkedListStack node and class outline
public class LinkedListStack<T> {
    private class Node {
        T data;
        Node next;
        Node(T data) { this.data = data; }
    }

    private Node top;    // head of the linked list = top of stack
    private int length;

    public LinkedListStack() {
        top    = null;
        length = 0;
    }
    ...
}

Push Operation

Create a new node, point its next to the current top, then update top to the new node.

push()
public void push(T item) {
    Node newNode = new Node(item);
    newNode.next = top;   // new node points to old top
    top = newNode;        // new node becomes the new top
    length++;
}

Pop Operation

Save the top node's data, advance top to the next node, and return the saved data.

pop()
public T pop() {
    if (isEmpty()) throw new RuntimeException("Stack is empty.");
    T data = top.data;   // save the value
    top = top.next;      // remove top node
    length--;
    return data;
}

Full Implementation and Demo

LinkedListStack.java
public class LinkedListStack<T> {
    private class Node {
        T data; Node next;
        Node(T d) { data = d; }
    }

    private Node top;
    private int length;

    public boolean isEmpty()  { return length == 0; }
    public int     getLength(){ return length; }

    public T peek() {
        if (isEmpty()) throw new RuntimeException("Stack is empty.");
        return top.data;
    }

    public void push(T item) {
        Node n = new Node(item);
        n.next = top;
        top = n;
        length++;
    }

    public T pop() {
        if (isEmpty()) throw new RuntimeException("Stack is empty.");
        T val = top.data;
        top   = top.next;
        length--;
        return val;
    }

    public static void main(String[] args) {
        LinkedListStack<Integer> stack = new LinkedListStack<>();
        stack.push(5);
        stack.push(10);
        stack.push(15);
        System.out.println("Peek: " + stack.peek());   // 15
        System.out.println("Pop:  " + stack.pop());    // 15
        System.out.println("Pop:  " + stack.pop());    // 10
        System.out.println("Size: " + stack.getLength()); // 1
    }
}
Output

Pop:  15
Pop:  10
Size: 1

Why Linked List vs Array?

  • A linked-list stack is naturally unbounded — it grows as long as heap memory is available.
  • An array-based stack has a fixed capacity (or must resize), but has better cache performance.
  • Both push and pop are O(1) for either implementation.
