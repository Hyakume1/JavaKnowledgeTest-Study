Exploring All Possibilities With Recursion

Recursion is amazing at exploring every single possibility — every ordering of a list, every combination of items. The pattern always looks the same:

  1. Make a choice (pick one item).
  2. Recurse with the remaining options.
  3. Undo the choice and try the next option.

All Orderings of a String — scrambleLetters()

A permutation is a different ordering of the same letters. "xy" has 2 permutations: xy and yx. To generate them all recursively, at each step you pick one letter from the unused letters (remainLetters), add it to your built-up word (scramLetters), recurse, then try the next letter.

scrambleLetters — generates all orderings
public static void scrambleLetters(String remainLetters, String scramLetters) {
    if (remainLetters.length() == 0) {   // BASE CASE: all letters placed
        System.out.println(scramLetters);
        return;
    }
    for (int i = 0; i < remainLetters.length(); i++) {
        String letter = remainLetters.substring(i, i + 1);
        String rest   = remainLetters.substring(0, i) + remainLetters.substring(i + 1);
        scrambleLetters(rest, scramLetters + letter);
    }
}

Trace for scrambleLetters("xy", ""):

  • i=0: pick "x" from "xy". Rest = "y". Recurse: scrambleLetters("y", "x") → prints "xy"
  • i=1: pick "y" from "xy". Rest = "x". Recurse: scrambleLetters("x", "y") → prints "yx"

Output: xy then yx — in that order.

All Fixed-Size Subsets — shoppingBagPermutations()

Sometimes you want to pick a fixed number of items from a larger set. For example: pick 3 items from a list of 4 (Milk, Belt, Toys, Cups). The method shoppingBagPermutations() does this.

shoppingBagPermutations — picks MAX_SHOPPING_BAG_SIZE items from remainingItems
static final int MAX_SHOPPING_BAG_SIZE = 3;

public static void shoppingBagPermutations(ArrayList<String> remainingItems,
                                            ArrayList<String> currBag) {
    if (currBag.size() == MAX_SHOPPING_BAG_SIZE) {  // BASE CASE: bag is full
        System.out.println(currBag);
        return;
    }
    for (int i = 0; i < remainingItems.size(); i++) {
        String item = remainingItems.remove(i);  // pick item from remaining
        currBag.add(item);                       // add to bag
        shoppingBagPermutations(remainingItems, currBag);  // recurse
        currBag.remove(currBag.size() - 1);      // undo: remove from bag
        remainingItems.add(i, item);             // undo: put back in remaining
    }
}

// main() calls it like this:
// possibleItems = [Milk, Belt, Toys, Cups]
// shoppingBagPermutations(possibleItems, new ArrayList<>());

Tracing shoppingBagPermutations

When main() first calls the method:

  • remainingItems = [Milk, Belt, Toys, Cups] — all 4 items, nothing picked yet
  • currBag = [] — the bag is empty, nothing chosen yet

What happens first after main() calls it? The bag isn't full (size 0 ≠ 3), so we skip the base case. The loop picks one item from remainingItems, adds it to currBag, and makes the first recursive call.

How many recursive calls before the first permutation prints? We need 3 items in the bag before the base case triggers. Each recursive call adds one item. So it takes 3 recursive calls to fill the bag — then the base case fires and prints the first combination.

Just before returning to main(): Every time a recursive branch finishes, it undoes its choices (removes from bag, puts back in remaining). So when we return all the way back to main(), all 4 items are restored to remainingItems and the bag is empty again.

What If There Aren't Enough Items?

If main() only puts 2 items in possibleItems (say just Milk and Belt), but MAX_SHOPPING_BAG_SIZE = 3, then you can never fill the bag to size 3 — you'll always run out of items first. The base case currBag.size() == 3 is never reached, so nothing gets printed.

Which Method to Use for Fixed-Size Subsets?

If you want to generate all possible 3-letter subsets from the letters in an N-letter word (N > 3), use shoppingBagPermutations — it's designed for picking a fixed number of items from a larger set. scrambleLetters generates ALL orderings of ALL letters, which is different.
