What Is a Stack?

A stack is a collection of items where you can only add or remove things from the top. The last item you put in is the first item that comes back out — this is called LIFO (Last-In, First-Out).

Think of a stack of plates in a cafeteria. You always place new plates on top, and you always grab plates from the top. You can't grab a plate from the middle without removing the ones above it first.

The Four Main Stack Operations

  • push(item) — puts an item on top of the stack
  • pop() — removes and returns the top item
  • peek() — looks at the top item WITHOUT removing it
  • isEmpty() — returns true if the stack has no items
  • getLength() — returns how many items are in the stack
Stack operations illustrated
// Start: stack is empty  []
push(7)    // stack: [7]           top = 7
push(14)   // stack: [7, 14]       top = 14
push(9)    // stack: [7, 14, 9]    top = 9
peek()     // returns 9, stack unchanged: [7, 14, 9]
pop()      // returns 9, stack: [7, 14]
pop()      // returns 14, stack: [7]
isEmpty()  // returns false
pop()      // returns 7, stack: []
isEmpty()  // returns true

Using Java's Built-In Stack

Java has a built-in class called ArrayDeque that works great as a stack. (There's also an older Stack class, but ArrayDeque is the modern preferred way.)

Using ArrayDeque as a stack
import java.util.ArrayDeque;
import java.util.Deque;

public class StackDemo {
    public static void main(String[] args) {
        Deque<Integer> stack = new ArrayDeque<>();

        stack.push(10);
        stack.push(20);
        stack.push(30);

        System.out.println("Top: " + stack.peek());  // 30
        System.out.println("Pop: " + stack.pop());   // 30
        System.out.println("Pop: " + stack.pop());   // 20
        System.out.println("Size: " + stack.size()); // 1
    }
}
Output 
Top: 30
Pop: 30
Pop: 20
Size: 1

Where Are Stacks Used in Real Life?

  • Undo button in a text editor — each action you take is pushed onto a stack. When you press Undo, the most recent action is popped off and reversed.
  • Browser back button — each page you visit is pushed. The back button pops the most recent page.
  • Java's own call stack — every time a method is called, Java pushes a frame onto an internal stack. When the method returns, it pops the frame off. This is literally how recursion works.
  • Checking matching parentheses — push each opening bracket, and pop when you see a closing bracket to check they match.
