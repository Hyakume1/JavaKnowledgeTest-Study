Stack Abstract Data Type

A stack is a collection of items where insertions and removals both happen at the same end, called the top. The last item pushed in is always the first one popped out — this is called LIFO (Last-In, First-Out).

Think of a stack of plates: you always add or remove plates from the top. You cannot access a plate in the middle without first removing the ones above it.

Core Stack Operations

  • push(item) — adds item to the top of the stack
  • pop() — removes and returns the top item
  • peek() — returns the top item without removing it
  • isEmpty() — returns true if the stack has no items
  • getLength() — returns the number of items
Stack operations illustrated
// Start: stack is empty  []
push(7)    // stack: [7]           top = 7
push(14)   // stack: [7, 14]       top = 14
push(9)    // stack: [7, 14, 9]    top = 9
peek()     // returns 9, stack unchanged: [7, 14, 9]
pop()      // returns 9, stack: [7, 14]
pop()      // returns 14, stack: [7]
isEmpty()  // returns false
pop()      // returns 7, stack: []
isEmpty()  // returns true

Using Java's Built-In Stack

Java's java.util.Stack class (and the preferred java.util.Deque used as a stack) provide a ready-made stack implementation.

Using ArrayDeque as a stack
import java.util.ArrayDeque;
import java.util.Deque;

public class StackDemo {
    public static void main(String[] args) {
        Deque<Integer> stack = new ArrayDeque<>();

        stack.push(10);
        stack.push(20);
        stack.push(30);

        System.out.println("Top: " + stack.peek());  // 30
        System.out.println("Pop: " + stack.pop());   // 30
        System.out.println("Pop: " + stack.pop());   // 20
        System.out.println("Size: " + stack.size()); // 1
    }
}
Output 
Top: 30
Pop: 30
Pop: 20
Size: 1

Real-World Uses of Stacks

  • Undo/redo in text editors — each action is pushed; undo pops.
  • Browser history — each visited page is pushed; the back button pops.
  • Method call tracking — the JVM's call stack is literally a stack of frames.
  • Balanced parenthesis checking — push each open bracket, pop when a close bracket is seen.
  • Evaluating expressions — calculators use stacks to handle operator precedence.
