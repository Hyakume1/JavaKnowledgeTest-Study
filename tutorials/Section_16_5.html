Why Create Custom Exception Types?

Catching Exception catches everything — including exceptions you didn't intend to handle. User-defined exception classes let you create specific, meaningful exception types so each catch block handles exactly one kind of problem.

Creating a Custom Exception

Extend Exception and call super(message) from the constructor to set the error message returned by getMessage().

Two custom exception classes
public class InvalidNegativeInputException extends Exception {
    public InvalidNegativeInputException(String varName) {
        super("Variable " + varName + " is negative.");
    }
}

public class NaNException extends Exception {
    public NaNException(String varName) {
        super("Variable " + varName + " is NaN (Not a Number).");
    }
}

Using Custom Exceptions: Density Calculator

Methods that may throw a checked exception must declare it with throws in the method signature. Callers then either handle it or propagate it further.

DensityCalculator.java — separate handlers for each error type
import java.util.Scanner;

public class DensityCalculator {

    public static double getPositiveValue(Scanner scnr, String name)
            throws InvalidNegativeInputException {
        System.out.print("Enter " + name + ": ");
        double val = scnr.nextDouble();
        if (val < 0.0) {
            throw new InvalidNegativeInputException(name);
        }
        return val;
    }

    public static double getDensity(Scanner scnr)
            throws InvalidNegativeInputException, NaNException {
        double mass   = getPositiveValue(scnr, "mass");
        double volume = getPositiveValue(scnr, "volume");
        double density = mass / volume;
        if (Double.isNaN(density)) {
            throw new NaNException("density");
        }
        return density;
    }

    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);
        try {
            System.out.printf("Density: %.4f%n", getDensity(scnr));
        } catch (InvalidNegativeInputException e) {
            System.out.println("Input error: " + e.getMessage());
        } catch (NaNException e) {
            System.out.println("Math error: " + e.getMessage());
        }
    }
}

Checked vs Unchecked Exceptions

  • Checked exceptions (extend Exception) — must be declared with throws or caught. Examples: IOException, FileNotFoundException, and your custom classes.
  • Unchecked exceptions (extend RuntimeException) — do not need to be declared. Examples: NullPointerException, ArrayIndexOutOfBoundsException, InputMismatchException.

Custom exceptions typically extend Exception (checked) when the caller is expected to handle the situation, and extend RuntimeException (unchecked) for programming errors that should never normally occur.
