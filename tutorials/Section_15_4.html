Reading Data From a File

So far you've read input that the user types. But what if all your data is already in a file? You can make your program read it automatically — no typing needed.

To read from a file, you need two things working together:

  1. A FileInputStream — connects to the file on disk. Think of it as opening the file and pointing at it.
  2. A Scanner wrapped around it — reads the contents using the same methods you already know.
The correct way to open and read a file
import java.util.Scanner;
import java.io.FileInputStream;
import java.io.IOException;

public static void main(String[] args) throws IOException {
    FileInputStream fbs = new FileInputStream("inputfile.txt");  // open the file
    Scanner inFS = new Scanner(fbs);                             // wrap it

    int x = inFS.nextInt();
    String s = inFS.next();

    fbs.close();  // always close when done
}

The Uninitialized Stream Bug

A very common mistake: you declare a FileInputStream variable but forget to actually open the file with new FileInputStream(...). If you try to wrap an uninitialized stream with Scanner, your program will crash.

Bug: stream declared but never opened
FileInputStream fbStream;        // declared only — NOT opened yet
Scanner inFS;
int numElem;
inFS = new Scanner(fbStream);   // ERROR! fbStream is uninitialized
numElem = inFS.nextInt();

The fix: assign the stream with fbStream = new FileInputStream("myfile.txt"); before wrapping it in a Scanner.

Reading Until the End of the File

You usually don't know exactly how many items are in a file. Use hasNext() or hasNextInt() in a loop:

  • hasNext() — returns true if there is any token left; returns false at the end of the file
  • hasNextInt() — returns true if the next token is an integer (does NOT consume it, just peeks)

Important: The loop runs once for every token in the file. It doesn't matter what you're searching for — while(inFS.hasNext()) always reads ALL tokens. A file with 7 tokens = 7 loop iterations, every time.

Counting word occurrences — loop runs for ALL 7 tokens
// File contents: twenty associable twenty unredacted associable folksay twenty
// (7 tokens)

String userWord = "associable";
int count = 0;

while (inFS.hasNext()) {           // runs 7 times — checks every token
    if (inFS.next().equals(userWord)) {
        count++;
    }
}
System.out.println(count);         // → 2

Reading Full Lines

next() reads one word at a time. nextLine() reads everything up to the end of the current line, including spaces.

nextLine() for multi-word fields
String lineString = inFS.nextLine();   // reads the whole line at once

Example: Tracking Average Ratings

To compute an average, you need: an int for the running sum, an int for the count, and a double for the final result (since dividing two ints truncates the decimal).

Three variables for average tracking
int userRatingSum   = 0;
int userRatingCount = 0;
double averageRating;

while (inFS.hasNext()) {
    String user   = inFS.next();
    int    rating = inFS.nextInt();
    userRatingSum   += rating;
    userRatingCount++;
}

averageRating = (double) userRatingSum / userRatingCount;
System.out.printf("Average: %.2f%n", averageRating);

What Triggers a FileNotFoundException?

If the file you specify doesn't exist on disk, Java throws a FileNotFoundException the moment you call new FileInputStream("filename"). That's why main() needs throws IOException — it lets the exception bubble up rather than crashing silently.
