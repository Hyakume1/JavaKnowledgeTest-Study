Building a Stack With an Array

Instead of a linked list, you can implement a stack using a simple array. The bottom of the stack lives at position 0, and the top is at the last filled position. A variable called length tracks how many items are currently in the stack.

Visual layout
// After push(5), push(10), push(15):
//  index:  0    1    2
//  array: [5,  10,  15,  ...]
//  length = 3        ^  top is here

Two Flavors: Bounded and Unbounded

  • A bounded stack has a max size. If you try to push when it's full, it throws an exception — no automatic growing.
  • An unbounded stack automatically doubles its array size when it gets full, so you can (theoretically) always push more items.

Note: The bounded stack's initial array length doesn't have to equal its maximum length. They might start smaller and grow up to maxLength. So "a bounded stack's initial size always equals maxLength" is FALSE.

When Does Resize Happen?

Push checks length == array.length — if the array is completely full, the unbounded stack doubles it. The bounded stack throws an exception instead of resizing.

Push logic — resize trigger
public boolean push(T item) {
    if (maxLength >= 0 && length == maxLength) {
        return false;  // bounded and FULL — no push (may throw exception)
    }
    if (length == array.length) {
        // Unbounded: need more room — DOUBLE the array
        array = Arrays.copyOf(array, array.length * 2);  // O(n) resize!
    }
    array[length] = item;
    length++;
    return true;
}

The resize only triggers when length == array.length (not before). Resize doubles the array: 10 → 20 → 40, etc. In reality, the unbounded stack can grow until the system runs out of memory — then resize() fails to allocate, and the stack can no longer grow.

The Constructor — Bounded or Unbounded?

One class, two modes
ArrayStack<Integer> bounded   = new ArrayStack<>(10); // bounded, max 10
ArrayStack<Integer> unbounded = new ArrayStack<>();    // NO argument = unbounded
ArrayStack<Integer> also_unb  = new ArrayStack<>(-1); // negative maxLength = unbounded

Calling the constructor without an argument creates an unbounded stack. The maxLength is set to -1 (which means "no limit"). An unbounded stack with maxLength = -1 can never have length == maxLength (since length is always ≥ 0 and maxLength is -1), so push never hits the full-check.

Pop — One-Liner Version

The standard two-line pop:

Two-line pop vs one-liner pop
// Standard two-line version:
T val = (T) array[length - 1];
length--;
return val;

// One-liner (equivalent):
return array[--length];   // pre-decrement: length becomes length-1, THEN access array

Key detail: Use --length (pre-decrement), NOT length-- (post-decrement). --length decrements first, then accesses the array at the new value — which is the old top position. length-- would access at the old length first (out of bounds!), then decrement.

Pop requires the stack length to be ≥ 1. Calling pop on an empty stack (length = 0) throws a "Stack underflow" exception.

Pop Never Reallocates

Pop only decrements the length counter. It does NOT change the array size or reallocate memory. The value is still in memory but becomes inaccessible (and will be overwritten by the next push). So: array-based stack implementations do NOT reallocate when popping.

Complexity

  • Bounded stack push/pop: always O(1) — no resize ever happens
  • Unbounded stack push: usually O(1), but O(n) worst case when a resize copies all n elements
  • Unbounded stack pop: always O(1) — just decrements length

Only the bounded stack guarantees O(1) worst-case for both push and pop. The unbounded stack only guarantees O(1) amortized (on average over many operations).

ArrayStack Implementation

ArrayStack.java
import java.util.Arrays;

public class ArrayStack<T> {
    private Object[] array;
    private int length;
    private int maxLength;  // negative = unbounded

    public ArrayStack(int maxLength) {
        int initialCap = (maxLength < 0) ? 10 : maxLength;
        array     = new Object[initialCap];
        length    = 0;
        this.maxLength = maxLength;
    }

    public ArrayStack() { this(-1); }  // no-arg constructor → unbounded

    public boolean isEmpty()   { return length == 0; }
    public int     getLength() { return length; }

    @SuppressWarnings("unchecked")
    public T peek() {
        if (isEmpty()) throw new RuntimeException("Stack underflow.");
        return (T) array[length - 1];
    }

    public boolean push(T item) {
        if (maxLength >= 0 && length == maxLength) return false;
        if (length == array.length) array = Arrays.copyOf(array, array.length * 2);
        array[length++] = item;
        return true;
    }

    @SuppressWarnings("unchecked")
    public T pop() {
        if (isEmpty()) throw new RuntimeException("Stack underflow.");
        return (T) array[--length];   // one-liner: pre-decrement then access
    }
}
