How Scanner Reads Words

When you call next() on a Scanner, it skips any spaces/tabs/newlines, then reads everything up to the next space/tab/newline. Punctuation is NOT automatically stripped — commas, periods, etc. become part of the token.

Reading One Character at a Time — useDelimiter("")

Pass an empty string to useDelimiter("") and now every call to next() returns exactly one character. This is useful for character counting or processing text character by character.

Character-by-character reading — counting every character
// Input: "A cup of Java."  (14 characters total)
data.useDelimiter("");   // each next() = one character
int count = 0;
while (data.hasNext()) {
    String input = data.next();  // one character at a time
    count++;
}
System.out.println(count);   // → 14
Why 14? Count every character: 
A   (1)
    (2) ← space
c   (3)
u   (4)
p   (5)
    (6) ← space
o   (7)
f   (8)
    (9) ← space
J   (10)
a   (11)
v   (12)
a   (13)
.   (14) ← period
Total: 14 characters

To count only non-letter characters: "A cup of Java." has 3 spaces + 1 period = 4 non-letter characters.

Counting non-letter characters using Character.isLetter()
data.useDelimiter("");
int count = 0;
while (data.hasNext()) {
    char input = data.next().charAt(0);
    if (!Character.isLetter(input)) { count++; }  // count spaces + punctuation
}
System.out.println(count);   // → 4  (3 spaces + 1 period)

Splitting Country Names and Populations

Imagine a file where each line has a country name and a population number, like:

China 1330044605
United States 303824646

The tricky part: country names can have spaces! Here are three approaches, and one right way:

Approach 1: Split on FIRST space (WRONG for multi-word names)

WRONG — first-space split fails for "United States"
int i = line.indexOf(' ');                   // finds FIRST space
String country = line.substring(0, i);       // "United"  ← WRONG!
String pop     = line.substring(i + 1);      // "States 303824646" ← wrong!

This breaks for "United States 303824646" — it treats "United" as the country and "States 303824646" as the population.

Approach 2: Split on LAST space with substring(i) (WRONG — leading space)

WRONG — substring(i) includes leading space
int i = line.lastIndexOf(' ');               // finds LAST space
String country = line.substring(0, i).trim(); // "United States" ← correct
String pop     = line.substring(i);          // " 303824646" ← has leading space!
// Integer.parseInt(" 303824646") throws NumberFormatException!

Approach 3: Split on LAST space with substring(i+1) (CORRECT)

CORRECT — lastIndexOf + substring(i+1)
int i = line.lastIndexOf(' ');               // finds LAST space
String country = line.substring(0, i).trim(); // "United States" ✓
String pop     = line.substring(i + 1);      // "303824646"    ✓ no leading space!

Approach 4: Use next() while !hasNextInt() (CORRECT, elegant)

CORRECT — read words until the number, then nextInt()
String country = "";
while (!in.hasNextInt()) {
    country += in.next() + " ";  // accumulate words until we see an integer
}
int pop = in.nextInt();          // now read the number
country = country.trim();        // remove trailing space

This works for any country name, no matter how many words it has!

The nextInt() + nextLine() Pitfall

This is one of the most common Scanner bugs. When you call nextInt(), it reads the number but leaves the newline character (\n) sitting in the buffer. The next nextLine() call reads that leftover newline — and returns an empty string instead of the next line.

Scenario A: Number and text on the SAME line

Input: "2 United States" (one line)
int n = in.nextInt();           // reads: 2. Buffer has: " United States\n"
String s = in.nextLine();       // reads " United States" (with leading space!)
// s = " United States" ← note the leading space

nextLine() returns " United States" — WITH a leading space. Not "United States".

Scenario B: Number and text on DIFFERENT lines

Input: "2" on line 1, "United States" on line 2
int n = in.nextInt();           // reads: 2. Buffer has: "\n" (just the newline)
String s = in.nextLine();       // reads "" ← empty string! (the leftover \n)
// "United States" is still waiting in the buffer!

After nextInt() reads 2, the buffer holds \n. nextLine() reads that \n and returns an empty string. You need a second nextLine() to actually get "United States".

The fix: consume the leftover newline
int n = in.nextInt();
in.nextLine();                  // ← throw away leftover newline
String s = in.nextLine();       // now reads "United States" ✓

Can you mix nextInt/nextDouble with nextLine? YES — you just need the extra nextLine() call to discard the newline. You should NOT avoid mixing them altogether.

Can you avoid the pitfall entirely? YES — read everything with nextLine(), then parse numbers from each string yourself:

Avoiding the pitfall by reading all lines first
String line = in.nextLine();         // "2 United States"
String[] parts = line.split(" ", 2); // ["2", "United States"]
int n    = Integer.parseInt(parts[0]);
String s = parts[1];

Testing What Kind of Character You Have

  • Character.isLetter(ch) — true if the character is a letter (a-z or A-Z)
  • Character.isDigit(ch) — true if the character is '0' through '9'
  • Character.isUpperCase(ch) — true if the character is an uppercase letter
  • Character.isLowerCase(ch) — true if the character is a lowercase letter
  • Character.isWhitespace(ch) — true if the character is a space, tab, or newline
