What Are Command-Line Arguments?

When you run a Java program from the terminal, you can pass extra information right after the program name. These pieces of information are called command-line arguments and they are available inside main() through the args array.

Think of it like calling a friend and saying "Hey, can you bring snacks AND drinks?" — you're passing extra instructions when you make the call.

Running a program with arguments
java MyProgram -v input.dat output.dat
//             ^    ^          ^
//          args[0] args[1]  args[2]

By convention, arguments starting with - are treated as flags (options that switch features on or off), while the others are usually file names or values. Your code decides how to interpret them.

Checking args inside main()
public static void main(String[] args) {
    if (args.length < 2) {
        System.out.println("Usage: java MyProgram infile outfile");
        return;
    }
    String inFile  = args[0];
    String outFile = args[1];
    System.out.println("Input:  " + inFile);
    System.out.println("Output: " + outFile);
}

Always check args.length before accessing any element! If you try to access args[0] when no arguments were given, your program will crash with ArrayIndexOutOfBoundsException.

Command-line arguments are great for automation — instead of typing input every time the program runs, you can script it to process hundreds of files automatically.

Full Example: Caesar Cipher

A Caesar cipher is a simple letter-shifting secret code: each letter is replaced by the letter a fixed number of positions forward in the alphabet. With a shift of 3, 'A' becomes 'D', 'B' becomes 'E', 'Z' wraps around to 'C'. To decrypt, shift backwards by the same amount.

This program reads a file character by character, shifts each letter, and writes the result to an output file. Command-line arguments specify the input and output file names, plus an optional -d flag to decrypt instead of encrypt.

Usage examples
java CaesarCipher input.txt encrypt.txt      // encrypt
java CaesarCipher -d encrypt.txt output.txt  // decrypt
CaesarCipher.java
import java.io.File;
import java.io.FileNotFoundException;
import java.io.PrintWriter;
import java.util.Scanner;

public class CaesarCipher {
    public static void main(String[] args) throws FileNotFoundException {
        final int DEFAULT_KEY = 3;
        int key = DEFAULT_KEY;
        String inFile = "", outFile = "";
        int fileCount = 0;

        // Parse command-line arguments
        for (String arg : args) {
            if (arg.startsWith("-")) {
                if (arg.equals("-d")) { key = -key; }  // decrypt: shift backwards
                else { usage(); return; }
            } else {
                fileCount++;
                if (fileCount == 1) inFile  = arg;
                else                outFile = arg;
            }
        }

        if (fileCount != 2) { usage(); return; }

        // Read input one character at a time; write encrypted output
        Scanner in = new Scanner(new File(inFile));
        in.useDelimiter(""); // each next() = one character
        PrintWriter out = new PrintWriter(outFile);

        while (in.hasNext()) {
            char from = in.next().charAt(0);
            out.print(encrypt(from, key));
        }

        in.close();
        out.close();
        System.out.println("Done. Output written to " + outFile);
    }

    /**
     * Shifts a letter by 'key' positions in the alphabet (wrapping around).
     * Non-letter characters are returned unchanged.
     */
    public static char encrypt(char ch, int key) {
        int base;
        if      ('A' <= ch && ch <= 'Z') base = 'A';
        else if ('a' <= ch && ch <= 'z') base = 'a';
        else return ch;  // not a letter — leave it alone

        final int LETTERS = 26;
        int offset = (ch - base + key % LETTERS + LETTERS) % LETTERS;
        return (char) (base + offset);
    }

    public static void usage() {
        System.out.println("Usage: java CaesarCipher [-d] infile outfile");
    }
}
Example: encrypting "Hello, World!" 
input.txt:   Hello, World!
encrypt.txt: Khoor, Zruog!

After decryption:
output.txt:  Hello, World!

Key Takeaways

  • Always check args.length before accessing any element — avoid the ArrayIndexOutOfBoundsException crash.
  • useDelimiter("") turns Scanner into a character-by-character reader — each next() call returns one character.
  • Non-letter characters (spaces, punctuation, numbers) are passed through unchanged by encrypt() — only letters get shifted.
  • The math % LETTERS + LETTERS) % LETTERS handles wrapping correctly for both positive (encrypt) and negative (decrypt) shift values.
