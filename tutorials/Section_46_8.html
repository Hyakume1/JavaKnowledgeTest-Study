Command-Line Arguments

When you run a Java program from the terminal, you can pass extra information after the class name. These are called command-line arguments and they arrive in the args array of main().

Running a program with arguments
java MyProgram -v input.dat output.dat
//             ^    ^          ^
//          args[0] args[1]  args[2]

Strings starting with - are conventionally treated as flags (options). The others are typically file names or values. Your code decides how to interpret them.

Checking args inside main()
public static void main(String[] args) {
    if (args.length < 2) {
        System.out.println("Usage: java MyProgram infile outfile");
        return;
    }
    String inFile  = args[0];
    String outFile = args[1];
    System.out.println("Input:  " + inFile);
    System.out.println("Output: " + outFile);
}

Interactive vs command-line: Prompting the user works well for infrequent use. Command-line arguments shine when you need to automate the program — you can script it to run hundreds of files unattended.

Caesar Cipher — Full Example

The Caesar cipher is a simple letter-shifting encryption: each letter is replaced by the letter K positions forward in the alphabet (wrapping around). With the default key of 3, 'A' → 'D', 'B' → 'E', 'Z' → 'C'. Decryption shifts backwards by K.

This program reads characters one at a time using useDelimiter(""), encrypts each letter, and writes the result to an output file. Command-line arguments specify input/output files and an optional -d flag for decryption.

Usage examples
java CaesarCipher input.txt encrypt.txt      // encrypt
java CaesarCipher -d encrypt.txt output.txt  // decrypt
CaesarCipher.java
import java.io.File;
import java.io.FileNotFoundException;
import java.io.PrintWriter;
import java.util.Scanner;

public class CaesarCipher {
    public static void main(String[] args) throws FileNotFoundException {
        final int DEFAULT_KEY = 3;
        int key = DEFAULT_KEY;
        String inFile = "", outFile = "";
        int fileCount = 0;

        // Parse command-line arguments
        for (String arg : args) {
            if (arg.startsWith("-")) {
                if (arg.equals("-d")) { key = -key; }  // decrypt: shift backwards
                else { usage(); return; }
            } else {
                fileCount++;
                if (fileCount == 1) inFile  = arg;
                else                outFile = arg;
            }
        }

        if (fileCount != 2) { usage(); return; }

        // Read input one character at a time; write encrypted output
        Scanner in = new Scanner(new File(inFile));
        in.useDelimiter(""); // each next() = one character
        PrintWriter out = new PrintWriter(outFile);

        while (in.hasNext()) {
            char from = in.next().charAt(0);
            out.print(encrypt(from, key));
        }

        in.close();
        out.close();
        System.out.println("Done. Output written to " + outFile);
    }

    /**
     * Shifts a letter by 'key' positions in the alphabet (wrapping around).
     * Non-letter characters are returned unchanged.
     */
    public static char encrypt(char ch, int key) {
        int base;
        if      ('A' <= ch && ch <= 'Z') base = 'A';
        else if ('a' <= ch && ch <= 'z') base = 'a';
        else return ch;  // not a letter — leave it alone

        final int LETTERS = 26;
        int offset = (ch - base + key % LETTERS + LETTERS) % LETTERS;
        return (char) (base + offset);
    }

    public static void usage() {
        System.out.println("Usage: java CaesarCipher [-d] infile outfile");
    }
}
Example: encrypting "Hello, World!" 
input.txt:   Hello, World!
encrypt.txt: Khoor, Zruog!

After decryption:
output.txt:  Hello, World!

Key Takeaways

  • args is always an array — check args.length before accessing any element to avoid ArrayIndexOutOfBoundsException.
  • useDelimiter("") turns the Scanner into a character-by-character reader.
  • Non-letter characters (spaces, punctuation, digits) are left unchanged by encrypt() — only letters are shifted.
  • The % LETTERS and + LETTERS in the offset calculation ensures correct wrapping for both positive (encrypt) and negative (decrypt) keys.
