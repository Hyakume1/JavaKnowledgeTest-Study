Binary Search With Recursion

Binary search finds a target in a sorted list by cutting the search in half each time. It's much faster than checking every item one by one. And because "search this half of the list" is the same problem applied to a smaller range, it maps perfectly to recursion.

Think of it like a guessing game: I'm thinking of a number from 1 to 100. You guess 50. I say "lower." Now you only need to search 1–49. You guess 25. I say "higher." Now 26–49. You keep cutting the range in half until you find it.

Guessing Game Example

NumberGuessGame.java
import java.util.Scanner;

public class NumberGuessGame {
    public static void guessNumber(int low, int high, Scanner scnr) {
        int mid = (low + high) / 2;
        System.out.print("Is it " + mid + "? (l=lower / h=higher / other=correct): ");
        char ans = scnr.next().charAt(0);

        if (ans != 'l' && ans != 'h') {   // base case: we found it!
            System.out.println("Got it!");
        } else if (ans == 'l') {           // search the lower half
            guessNumber(low, mid, scnr);
        } else {                           // search the upper half
            guessNumber(mid + 1, high, scnr);
        }
    }

    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);
        System.out.println("Think of a number from 0 to 100.");
        guessNumber(0, 100, scnr);
    }
}

Binary Search on a Sorted ArrayList

The findMatch() method below searches within the index range from low to high. Each call checks the middle item. If it matches, return the index. If there's only one item left and it doesn't match, return -1. Otherwise, recurse into the correct half.

NameFinder.java
import java.util.ArrayList;
import java.util.Scanner;

public class NameFinder {
    public static int findMatch(ArrayList<String> list, String target,
                                int low, int high) {
        int mid = (low + high) / 2;
        int range = (high - low) + 1;

        if (target.equals(list.get(mid))) {           // base case 1: found it!
            return mid;
        } else if (range == 1) {                       // base case 2: not in list
            return -1;
        } else if (target.compareTo(list.get(mid)) < 0) {
            return findMatch(list, target, low, mid);  // search lower half
        } else {
            return findMatch(list, target, mid + 1, high); // search upper half
        }
    }

    public static void main(String[] args) {
        ArrayList<String> attendees = new ArrayList<String>();
        attendees.add("Adams, Mary");
        attendees.add("Carver, Michael");
        attendees.add("Domer, Hugo");
        attendees.add("Fredericks, Carlos");
        attendees.add("Li, Jie");

        Scanner scnr = new Scanner(System.in);
        System.out.print("Enter name (Last, First): ");
        String name = scnr.nextLine();

        int pos = findMatch(attendees, name, 0, attendees.size() - 1);
        if (pos >= 0) {
            System.out.println("Found at position " + pos + ".");
        } else {
            System.out.println("Not found.");
        }
    }
}

Why Is Binary Search So Fast?

Each recursive call cuts the search range in half. So for a list of 1,000 names, you'd need at most 10 steps to find any name (because 2^10 = 1,024). Checking every item one by one would take up to 1,000 steps. The bigger the list, the more dramatic the difference.

Important requirement: The list must be sorted for binary search to work. If the list is not sorted, binary search will give wrong answers.
