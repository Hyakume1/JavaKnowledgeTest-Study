What Is a Queue?

A queue is a collection where items enter at the back and leave from the front — just like a line of people waiting at a store. The first person who got in line is the first person served. This is called FIFO (First-In, First-Out).

A stack is LIFO — last in, first out (like a pile of plates). A queue is the opposite — FIFO — first in, first out (like a checkout line). Both are useful, just for different situations.

The Four Main Queue Operations

  • enqueue(item) — adds an item to the back of the line
  • dequeue() — removes and returns the item at the front of the line
  • peek() — looks at the front item WITHOUT removing it
  • isEmpty() — returns true if the queue has no items
  • getLength() — returns how many items are in the queue

Important: calling dequeue() or peek() on an empty queue is an error — just like trying to serve someone when the line is empty.

Queue operations illustrated
// Start: queue is empty  []
enqueue(7)    // queue: [7]             front=7
enqueue(14)   // queue: [7, 14]
enqueue(9)    // queue: [7, 14, 9]      back=9
peek()        // returns 7, queue unchanged
dequeue()     // returns 7, queue: [14, 9]
dequeue()     // returns 14, queue: [9]
isEmpty()     // returns false
dequeue()     // returns 9, queue: []
isEmpty()     // returns true

Using Java's Built-In Queue

Java's LinkedList class can act as a queue. The method names are slightly different: offer() instead of enqueue, poll() instead of dequeue.

Queue demo with LinkedList
import java.util.LinkedList;
import java.util.Queue;

public class QueueDemo {
    public static void main(String[] args) {
        Queue<String> queue = new LinkedList<>();

        queue.offer("Alice");   // enqueue
        queue.offer("Bob");
        queue.offer("Carol");

        System.out.println("Front: " + queue.peek());    // Alice
        System.out.println("Remove: " + queue.poll());   // Alice (dequeue)
        System.out.println("Remove: " + queue.poll());   // Bob
        System.out.println("Size: "  + queue.size());    // 1
    }
}
Output 
Front: Alice
Remove: Alice
Remove: Bob
Size: 1

Stack vs Queue — Side by Side

  • Stack (LIFO): last in, first out — like a pile of books. Great for undo, backtracking, and recursion.
  • Queue (FIFO): first in, first out — like a waiting line. Great for task scheduling, breadth-first search, and processing events in order.

Where Are Queues Used in Real Life?

  • Print spooler — print jobs wait in a queue and are printed in the order they were sent.
  • Keyboard input — keys you press are queued up until the program reads them.
  • Breadth-first search (BFS) — a graph traversal algorithm that uses a queue to explore nodes level by level.
  • Thread pools — work tasks wait in a queue until a worker thread becomes available to handle them.
