Queue Abstract Data Type

A queue is a collection where items are added at one end (the back/rear) and removed from the other end (the front). The first item added is always the first item removed — FIFO (First-In, First-Out).

Think of a checkout line at a store: customers join at the back and leave from the front in the order they arrived.

Core Queue Operations

  • enqueue(item) — adds item to the back of the queue
  • dequeue() — removes and returns the item at the front
  • peek() — returns the front item without removing it
  • isEmpty() — returns true if the queue has no items
  • getLength() — returns the number of items

Important: calling dequeue() or peek() on an empty queue results in an error.

Queue operations illustrated
// Start: queue is empty  []
enqueue(7)    // queue: [7]             front=7
enqueue(14)   // queue: [7, 14]
enqueue(9)    // queue: [7, 14, 9]      back=9
peek()        // returns 7, queue unchanged
dequeue()     // returns 7, queue: [14, 9]
dequeue()     // returns 14, queue: [9]
isEmpty()     // returns false
dequeue()     // returns 9, queue: []
isEmpty()     // returns true

Using Java's Built-In Queue

Java's java.util.LinkedList implements the Queue interface and is the simplest ready-made queue.

Queue demo with LinkedList
import java.util.LinkedList;
import java.util.Queue;

public class QueueDemo {
    public static void main(String[] args) {
        Queue<String> queue = new LinkedList<>();

        queue.offer("Alice");   // enqueue
        queue.offer("Bob");
        queue.offer("Carol");

        System.out.println("Front: " + queue.peek());    // Alice
        System.out.println("Remove: " + queue.poll());   // Alice (dequeue)
        System.out.println("Remove: " + queue.poll());   // Bob
        System.out.println("Size: "  + queue.size());    // 1
    }
}
Output 
Front: Alice
Remove: Alice
Remove: Bob
Size: 1

Queue vs Stack

  • Stack (LIFO): last in, first out — like a pile of books. Great for undo, backtracking, recursion.
  • Queue (FIFO): first in, first out — like a waiting line. Great for task scheduling, BFS, buffering.

Real-World Queue Uses

  • Print spooler — print jobs are queued and processed in arrival order.
  • Keyboard input buffer — keystrokes are queued until the program reads them.
  • Breadth-first search (BFS) — graph traversal uses a queue to process nodes level by level.
  • Thread pools — work items wait in a queue for an available worker thread.
