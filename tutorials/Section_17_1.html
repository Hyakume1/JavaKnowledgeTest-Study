What Is Recursion?

A recursive method is a method that calls itself. Each call works on a smaller version of the same problem until it reaches a simple case that can be solved directly — the base case.

Think of it like looking up a word in a dictionary: if the definition uses another unfamiliar word, you look that up too, and so on, until you reach words you already know. Those known words are the base cases.

Anatomy of a Recursive Method

Every recursive method must have:

  1. Base case — a condition that stops the recursion and returns a value directly.
  2. Recursive case — reduces the problem and calls the method again on the smaller version.
Classic example: factorial
// 5! = 5 × 4 × 3 × 2 × 1 = 120
// Recursive definition: N! = N × (N-1)!,  base case: 0! = 1

public static int factorial(int n) {
    if (n == 0) {           // base case
        return 1;
    } else {                // recursive case
        return n * factorial(n - 1);
    }
}

Tracing the Recursive Calls

Calling factorial(4) expands like this before any result is returned:

Call stack for factorial(4)
factorial(4)
  → 4 * factorial(3)
        → 3 * factorial(2)
              → 2 * factorial(1)
                    → 1 * factorial(0)
                              → returns 1     (base case)
                    → returns 1 * 1 = 1
              → returns 2 * 1 = 2
        → returns 3 * 2 = 6
  → returns 4 * 6 = 24

Example: Countdown

Countdown.java
public class Countdown {
    public static void countDown(int n) {
        if (n == 0) {                     // base case
            System.out.println("Go!");
        } else {                          // recursive case
            System.out.println(n);
            countDown(n - 1);             // smaller problem
        }
    }

    public static void main(String[] args) {
        countDown(5);
    }
}
Output 
5
4
3
2
1
Go!

Common Mistakes

  • No base case — the method calls itself forever, causing a StackOverflowError.
  • Base case never reached — the recursive call does not actually reduce the problem toward the base case.
  • Missing return value — forgetting to return the result of the recursive call.
Infinite recursion — will crash
// BUG: no base case — n never changes
public static int bad(int n) {
    return n + bad(n);   // StackOverflowError!
}

Recursion vs Iteration

Any recursive solution can be rewritten as a loop, and vice versa. Recursion is preferred when the problem naturally breaks into self-similar subproblems (trees, binary search, permutations). Iteration is preferred when a simple counter controls repetition — it uses less memory and is faster.
