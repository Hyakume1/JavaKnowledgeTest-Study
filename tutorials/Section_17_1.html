What Is Recursion?

A recursive method is a method that calls itself. Sounds weird, right? Here's how to think about it: you have a big problem, so you break it into a slightly smaller version of the same problem, and keep breaking it down until you reach a problem so simple it has an obvious answer.

Imagine you're looking up what the word "recursion" means in a dictionary, and the definition uses another word you don't know. So you look up that word, and that definition uses another word you don't know. You keep looking up words until you find a word you already know — and that's your stopping point. Those known words are the base cases.

Every Recursive Method Needs Two Things

  1. Base case — the simple situation where the answer is obvious and you stop recursing. Without this, your method will call itself forever and crash.
  2. Recursive case — where you call the method again on a smaller version of the problem, moving closer to the base case.
Classic example: factorial (5! = 5 × 4 × 3 × 2 × 1 = 120)
// Recursive definition: N! = N × (N-1)!,  base case: 0! = 1

public static int factorial(int n) {
    if (n == 0) {           // base case — stop here
        return 1;
    } else {                // recursive case — break into smaller problem
        return n * factorial(n - 1);
    }
}

Tracing the Calls Step by Step

Let's trace what happens when you call factorial(4). Each line is a new call to the method, going deeper and deeper until the base case is hit, then the answers come back up:

Call chain for factorial(4)
factorial(4)
  → 4 * factorial(3)
        → 3 * factorial(2)
              → 2 * factorial(1)
                    → 1 * factorial(0)
                              → returns 1     (base case reached!)
                    → returns 1 * 1 = 1
              → returns 2 * 1 = 2
        → returns 3 * 2 = 6
  → returns 4 * 6 = 24

Example: Countdown

Countdown.java
public class Countdown {
    public static void countDown(int n) {
        if (n == 0) {                     // base case
            System.out.println("Go!");
        } else {                          // recursive case
            System.out.println(n);
            countDown(n - 1);             // call again with a smaller number
        }
    }

    public static void main(String[] args) {
        countDown(5);
    }
}
Output 
5
4
3
2
1
Go!

Common Mistakes

  • No base case — the method calls itself forever, which causes a StackOverflowError crash.
  • Base case is never reached — the recursive call doesn't actually make the problem smaller (e.g., accidentally passing the same value instead of a smaller one).
  • Forgetting to return the result — calling the method recursively but forgetting to return the value it gives back.
This will crash — infinite recursion
// BUG: n never gets smaller — will call itself forever!
public static int bad(int n) {
    return n + bad(n);   // StackOverflowError!
}

Recursion vs. Using a Loop

Any recursive method can also be written as a loop, and vice versa. Loops are usually simpler and faster for basic counting tasks like factorial. Recursion really shines when the problem naturally breaks into smaller pieces of the same shape — like searching, exploring all possibilities, or working with trees. Use the one that makes the code clearest.
