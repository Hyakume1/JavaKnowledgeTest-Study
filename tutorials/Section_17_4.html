Debugging Recursive Methods

Recursion can be hard to debug because many calls are happening at the same time, each with their own separate variables. It's like having 10 versions of the same function all running at once — confusing!

A very useful trick: add print statements that show what each call is doing. To make the output even more readable, indent each print statement based on how deep the recursion is. The deeper the call, the more indented the output — so you can visually see the tree of calls.

Adding an Indent Parameter

Pass an extra indent string through each recursive call. Each level adds more spaces to indent, so the deeper you go, the further right the output appears.

NameFinder.java with debug output
import java.util.ArrayList;
import java.util.Scanner;

public class NameFinder {
    public static int findMatch(ArrayList<String> list, String target,
                                int low, int high, String indent) {
        int mid   = (low + high) / 2;
        int range = (high - low) + 1;

        System.out.println(indent + "Searching range [" + low + ", " + high + "]"
                           + ", mid = " + mid + " (" + list.get(mid) + ")");

        int result;
        if (target.equals(list.get(mid))) {
            System.out.println(indent + "→ Found!");
            result = mid;
        } else if (range == 1) {
            System.out.println(indent + "→ Not found.");
            result = -1;
        } else if (target.compareTo(list.get(mid)) < 0) {
            System.out.println(indent + "→ Go lower.");
            result = findMatch(list, target, low, mid, indent + "   ");
        } else {
            System.out.println(indent + "→ Go higher.");
            result = findMatch(list, target, mid + 1, high, indent + "   ");
        }

        System.out.println(indent + "Returning " + result);
        return result;
    }

    public static void main(String[] args) {
        ArrayList<String> attendees = new ArrayList<>();
        attendees.add("Adams, Mary");
        attendees.add("Carver, Michael");
        attendees.add("Domer, Hugo");
        attendees.add("Fredericks, Carlos");
        attendees.add("Li, Jie");

        Scanner scnr = new Scanner(System.in);
        System.out.print("Enter name: ");
        String name = scnr.nextLine();

        int pos = findMatch(attendees, name, 0, attendees.size() - 1, "   ");
        System.out.println(pos >= 0 ? "Found at " + pos : "Not found.");
    }
}

Debugging Tips

  • Print the parameters at the start of each call — this confirms the problem is actually getting smaller with each call.
  • Print the return value right before you return it — this lets you trace values flowing back up the chain.
  • When debugging is done, comment out or remove the print statements. They also serve as documentation for how the recursion works.
  • If the recursion looks infinite, add a check: if the depth exceeds a reasonable number, print a warning and stop.
