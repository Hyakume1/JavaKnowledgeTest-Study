Writing to a File

Just like you can read data from a file, you can also save your program's output to a file. Instead of printing to the screen, you print to a file.

To write to a file, you need two things working together:

  1. A FileOutputStream — takes a String filename and connects to that file on disk.
  2. A PrintWriter wrapped around it — gives you familiar methods like println() and printf() that write to the file.
Basic write pattern
import java.io.FileOutputStream;
import java.io.PrintWriter;
import java.io.IOException;

FileOutputStream outputStream = new FileOutputStream("outfile.txt");  // open file
PrintWriter outFS = new PrintWriter(outputStream);                    // wrap it

outFS.println("Hello, file!");
outFS.printf("Value: %d%n", 42);

outFS.close();   // always close when done!

FileOutputStream Creates the File IMMEDIATELY

This surprises many beginners: the moment you call new FileOutputStream("myoutfile.txt"), the file is created on disk — even before you write a single byte to it!

File is created on open, not on write
fileStream = new FileOutputStream("myoutfile.txt");  // myoutfile.txt is created NOW
fileStream.close();  // file already exists on disk (empty, but it's there)

This also means: if the file cannot be created (bad path, no permission), FileOutputStream throws an exception right at that line.

PrintWriter Wraps Any OutputStream

The PrintWriter constructor takes one parameter: an OutputStream. It doesn't care what kind of OutputStream it is — as long as it's an OutputStream, PrintWriter can wrap it.

PrintWriter can wrap file output OR screen output
FileOutputStream outputStream = new FileOutputStream("outfile.txt");
PrintWriter outFS = new PrintWriter(outputStream);   // wraps a FileOutputStream

Here's the cool part: System.out is an object of the class PrintStream, and FileOutputStream is also an OutputStream. Both inherit from the same parent class (OutputStream), so PrintWriter can wrap either one!

The inheritance chain
OutputStream (parent)
  ├── FileOutputStream    ← used for writing to files
  └── PrintStream         ← what System.out actually is
                            (both are OutputStreams, so PrintWriter wraps either)

What Does PrintWriter Actually Do?

FileOutputStream can only write raw bytes. PrintWriter is the layer that lets you write all the normal Java types — ints, doubles, Strings, booleans — easily, using methods like println() and printf(). It converts everything to characters for you.

println() is a method of PrintWriter (for writing to files). You're probably used to using System.out.println() — that's also calling println(), but on a PrintStream object.

Always Close Your Files

Call close() on both your PrintWriter and FileOutputStream when you're done. Without closing, Java might not finish writing everything to the file, and your output could be truncated or missing.

Always close the stream
outFS.close();  // closes PrintWriter — also flushes any remaining buffered output
// fileStream.close() is also good practice

Summary Table

  • FileOutputStream("filename") — takes a String filename; creates the file immediately on disk; throws exception if can't open
  • new PrintWriter(outputStream) — takes an OutputStream; wraps it so you can use println() and printf()
  • PrintWriter.println() — writes a line to the file (same syntax you know from System.out)
  • close() — must be called when done; otherwise the file might be incomplete
