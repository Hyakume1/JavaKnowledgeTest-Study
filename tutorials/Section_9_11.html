What Is an ArrayList?

Imagine a shopping list on paper. A regular Java array is like writing your list on a piece of paper with exactly 5 lines — you can only ever have 5 items. An ArrayList is like a magical shopping list that automatically grows new lines when you add more items, and shrinks when you remove them. You never run out of space, and you never waste empty lines.

To use an ArrayList, you need to import it at the top of your file: import java.util.ArrayList;

How to create an ArrayList
ArrayList<Integer> nums  = new ArrayList<Integer>();
ArrayList<String>  names = new ArrayList<String>();

The type in the angle brackets (<Integer>, <String>) tells Java what kind of things you want to store. One important rule: you cannot use basic types like int or double — you have to use their "wrapper" versions: Integer, Double, etc.

The Most Important ArrayList Methods

Think of teamNums as an ArrayList that currently holds these numbers in order: 5, 9, 23, 11, 14.

Common operations
teamNums.size()         // → 5  (how many items are in the list)
teamNums.isEmpty()      // → false (is the list empty? No.)
teamNums.get(2)         // → 23 (get the item at position 2 — counting starts at 0!)
teamNums.set(0, 99)     // replaces the item at position 0 with 99, returns old value 5
teamNums.add(77)        // adds 77 at the very end of the list
teamNums.add(1, 88)     // inserts 88 at position 1, pushes everything else right
teamNums.remove(3)      // removes the item at position 3, pulls everything else left
teamNums.clear()        // removes ALL items; size → 0

Remember: positions (indexes) start at 0, not 1. So the first item is at position 0, the second is at position 1, and so on. If you try to access a position that doesn't exist, Java will crash with an IndexOutOfBoundsException.

Removing by Position vs. Removing by Value

This is a tricky gotcha! remove(3) removes the item at position 3 — not the item whose value is 3. If you want to remove the number 3 by its value, you have to write: list.remove(Integer.valueOf(3)). This tells Java "I mean the value 3, not position 3."

Looping Through an ArrayList

There are two common ways to go through every item in an ArrayList:

Two loop patterns
// Loop with index — use this when you need to know the position
for (int i = 0; i < teamNums.size(); i++) {
    System.out.println(teamNums.get(i));
}

// Simpler loop — use this when you only care about the values
for (int val : teamNums) {
    System.out.println(val);
}

Full Example: A Player Roster

Here is a complete program that lets you add players, remove players, and print the roster, all using an ArrayList:

PlayerManager.java
import java.util.ArrayList;
import java.util.Scanner;

public class PlayerManager {
    private ArrayList<Integer> players = new ArrayList<Integer>();

    public void addPlayer(int num) {
        players.add(num);
    }

    public void deletePlayer(int num) {
        for (int i = 0; i < players.size(); i++) {
            if (players.get(i).equals(num)) {
                players.remove(i);
                return;
            }
        }
    }

    public void printPlayers() {
        for (int n : players) {
            System.out.println(" " + n);
        }
    }

    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);
        PlayerManager roster = new PlayerManager();
        String cmd = "-";
        System.out.println("Commands: a=add  d=delete  p=print  q=quit");
        while (!cmd.equals("q")) {
            System.out.print("Command: ");
            cmd = scnr.next();
            if (cmd.equals("a")) {
                System.out.print("Player number: ");
                roster.addPlayer(scnr.nextInt());
            } else if (cmd.equals("d")) {
                System.out.print("Player number: ");
                roster.deletePlayer(scnr.nextInt());
            } else if (cmd.equals("p")) {
                roster.printPlayers();
            }
        }
    }
}

Quick Summary

  • ArrayList is like a regular array, but it grows and shrinks automatically.
  • Positions start at 0 — the first item is always at index 0.
  • Use wrapper types like Integer and Double, not int and double.
  • remove(3) removes by position — to remove by value, use remove(Integer.valueOf(3)).
