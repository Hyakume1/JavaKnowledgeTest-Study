What Is an ArrayList?

Imagine a shopping list on paper. A regular Java array is like writing your list on a piece of paper with exactly 5 lines — you can only ever have 5 items. An ArrayList is like a magical shopping list that automatically grows new lines when you add more items, and shrinks when you remove them.

To use an ArrayList, you need to import it at the top of your file: import java.util.ArrayList;

How to create an ArrayList
ArrayList<Integer> nums  = new ArrayList<Integer>();
ArrayList<String>  names = new ArrayList<String>();

The type in the angle brackets (<Integer>, <String>) tells Java what kind of things you want to store. You must use wrapper types: Integer instead of int, Double instead of double.

The Most Important ArrayList Methods

Think of itemList as holding: [33, 77, 44]

Common operations — notice the parentheses!
itemList.size()         // → 3  (how many items are in the list)
itemList.isEmpty()      // → false (is it empty? No.)
itemList.get(1)         // → 77  (spot 1 — counting starts at 0!)
itemList.add(55)        // adds 55 at the end → [33, 77, 44, 55]
itemList.add(1, 55)     // inserts 55 at spot 1 → [33, 55, 77, 44]
itemList.clear()        // removes ALL items; size → 0

Important — always write size() with parentheses! Writing itemList.size (without parentheses) is wrong — and itemList().size() is also wrong because itemList is a variable, not a method. The correct way is always itemList.size().

isEmpty() does NOT remove items! It just checks "is the list empty?" and returns true or false. To actually empty the list, use clear().

Positions start at 0! If you try get(8) on a list with only 8 items (spots 0–7), Java crashes with an IndexOutOfBoundsException. The last valid spot is always size() - 1.

Removing by Position vs. Removing by Value

This is a super tricky gotcha! On an ArrayList<Integer>, remove(55) removes the item at position 55 — not the item whose value is 55. If you want to remove the value 55, you must write: list.remove(Integer.valueOf(55)). This tells Java "I mean the value, not the index."

Position vs. Value — the remove() trap
ArrayList<Integer> itemList = new ArrayList<Integer>();
// itemList = [99, 33, 55, 77, 44]

itemList.remove(55);              // removes item AT INDEX 55 → ERROR! (only 5 items)
itemList.remove(Integer.valueOf(55)); // removes the VALUE 55 → [99, 33, 77, 44] ✓

Inserting at a Specific Position

When you use add(index, value), the new item is squeezed in at that spot and everything to its right shifts one position over.

add() with an index pushes items right
// itemList = [33, 77, 44]
itemList.add(1, 55);   // insert 55 at spot 1
// itemList = [33, 55, 77, 44]   ← 77 and 44 shifted right

itemList.add(0, 99);   // insert 99 at spot 0 (the very front)
// itemList = [99, 33, 55, 77, 44]

Keeping a List in Sorted Order

If you want to maintain a sorted list while inserting, you need to find the right spot first, then call add(index, newItem).

  • Ascending (low to high): insert the new item at the position of the first item that is greater than the new item. This keeps everything in order from smallest to biggest.
  • Descending (high to low): insert the new item at the position of the first item that is less than the new item. This keeps everything in order from biggest to smallest.
Keeping a list in ascending order
// We have a sorted ascending list: [10, 20, 40, 50]
// We want to insert 30.
// First item greater than 30 is 40, at index 2.
// So we insert at index 2.
itemList.add(2, 30);   // → [10, 20, 30, 40, 50] ✓
