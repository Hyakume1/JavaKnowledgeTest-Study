Throwing Your Own Exceptions

So far exceptions happened automatically when something went wrong. But you can also manually create and throw an exception yourself using the throw keyword. This is useful for input validation — checking if the user's input makes sense before you use it.

Think of it like a bouncer at a club: before anyone gets in, you check them. If they don't meet the requirements, you throw them out.

Basic throw syntax
throw new Exception("Descriptive error message here");

When throw runs, the method stops immediately and the exception travels up the call stack until something catches it.

Using throw to Validate Input

DensityCalculator.java
import java.util.Scanner;

public class DensityCalculator {
    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);

        try {
            System.out.print("Enter mass (kg): ");
            double mass = scnr.nextDouble();
            if (mass < 0.0) {
                throw new Exception("Invalid mass: cannot be negative.");
            }

            System.out.print("Enter volume (m^3): ");
            double volume = scnr.nextDouble();
            if (volume < 0.0) {
                throw new Exception("Invalid volume: cannot be negative.");
            }

            System.out.printf("Density: %.4f kg/m^3%n", mass / volume);

        } catch (Exception e) {
            System.out.println("Error: " + e.getMessage());
        }
    }
}

Catching Multiple Exception Types

Sometimes different things can go wrong, and you want to handle each one differently. You can add multiple catch blocks after a single try block. Java will check them in order from top to bottom and run the first one that matches.

BMICalculator.java — different handler for each error type
import java.util.Scanner;
import java.util.InputMismatchException;

public class BMICalculator {
    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);

        try {
            System.out.print("Enter weight (lbs): ");
            int weight = scnr.nextInt();
            if (weight < 0) throw new Exception("Invalid weight.");

            System.out.print("Enter height (in): ");
            int height = scnr.nextInt();
            if (height < 0) throw new Exception("Invalid height.");

            double bmi = ((double) weight / (height * height)) * 703.0;
            System.out.printf("BMI: %.2f%n", bmi);

        } catch (InputMismatchException e) {
            System.out.println("Input must be a whole number. Cannot compute BMI.");
        } catch (Exception e) {
            System.out.println(e.getMessage() + " Cannot compute BMI.");
        }
    }
}

Order of catch Blocks Matters!

Always put the more specific exception types first. Here's why: InputMismatchException is actually a special type of Exception. If you put catch (Exception e) first, it would catch everything — including InputMismatchException — and the specific catch block would never run.

Rule: specific exceptions go on top, general exceptions go on the bottom.
