Throwing Exceptions with throw

A program can deliberately signal an error using a throw statement. This is useful for validating inputs and keeping error-checking separate from normal logic.

Basic throw syntax
throw new Exception("Descriptive error message here");

When throw executes, the current method exits immediately and the exception propagates up the call stack until a catch block handles it.

Using throw for Input Validation

DensityCalculator.java
import java.util.Scanner;

public class DensityCalculator {
    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);

        try {
            System.out.print("Enter mass (kg): ");
            double mass = scnr.nextDouble();
            if (mass < 0.0) {
                throw new Exception("Invalid mass: cannot be negative.");
            }

            System.out.print("Enter volume (m^3): ");
            double volume = scnr.nextDouble();
            if (volume < 0.0) {
                throw new Exception("Invalid volume: cannot be negative.");
            }

            System.out.printf("Density: %.4f kg/m^3%n", mass / volume);

        } catch (Exception e) {
            System.out.println("Error: " + e.getMessage());
        }
    }
}

Multiple catch Blocks

A single try block can throw different exception types. Add multiple catch blocks to handle each type separately. Java checks them in order — the first matching type wins.

BMICalculator.java — multiple exception handlers
import java.util.Scanner;
import java.util.InputMismatchException;

public class BMICalculator {
    public static void main(String[] args) {
        Scanner scnr = new Scanner(System.in);

        try {
            System.out.print("Enter weight (lbs): ");
            int weight = scnr.nextInt();
            if (weight < 0) throw new Exception("Invalid weight.");

            System.out.print("Enter height (in): ");
            int height = scnr.nextInt();
            if (height < 0) throw new Exception("Invalid height.");

            double bmi = ((double) weight / (height * height)) * 703.0;
            System.out.printf("BMI: %.2f%n", bmi);

        } catch (InputMismatchException e) {
            System.out.println("Input must be a whole number. Cannot compute BMI.");
        } catch (Exception e) {
            System.out.println(e.getMessage() + " Cannot compute BMI.");
        }
    }
}
Three sample runs

Enter height (in): 66
BMI: 24.21

---

---

Order Matters for catch Blocks

Always list more specific exception types before more general ones. InputMismatchException is a subclass of Exception, so it must come first. If catch (Exception e) appeared first, it would swallow all exceptions including InputMismatchException.
